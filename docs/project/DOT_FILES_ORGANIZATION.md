
## ğŸ›¡ï¸ Security Best Practices

### âœ… DO

1. **Copy templates, don't edit**:
   ```bash
   cp .env.example .env  # âœ… Good
   # NOT: mv .env.example .env  # âŒ Bad
   ```

2. **Use secrets managers**:
   - AWS Secrets Manager
   - HashiCorp Vault
   - Azure Key Vault
   - Google Secret Manager

3. **Rotate credentials**:
   - Every 90 days minimum
   - After team member changes
   - After suspected breaches

4. **Use strong passwords**:
   - Min 32 characters
   - Random generation
   - No dictionary words

5. **Validate on startup**:
   ```python
   # Check required vars exist
   required = ['DB_HOST', 'DB_PASSWORD']
   missing = [v for v in required if not os.getenv(v)]
   if missing:
       raise ValueError(f"Missing: {missing}")
   ```

### âŒ DON'T

1. **Don't commit `.env`**:
   ```bash
   # Check before committing
   git status
   # Should NOT see .env
   ```

2. **Don't share `.env` files**:
   - Use `.env.example` instead
   - Each developer has their own

3. **Don't hardcode secrets**:
   ```python
   # âŒ Bad
   password = "admin123"
   
   # âœ… Good
   password = os.getenv('DB_PASSWORD')
   ```

4. **Don't log secrets**:
   ```python
   # âŒ Bad
   logger.info(f"Password: {password}")
   
   # âœ… Good
   logger.info("Connected to database")
   ```
